# Creating self-signed certificates on Android
########################################################
Step 1 Create private keys
> openssl genrsa -des3 -out client_key.pem 2048
> openssl genrsa -des3 -out server_key.pem 2048
Step 2. Create self-signed certificates
> openssl req -new -x509 -key client_key.pem -out client.pem -days 3650
> openssl req -new -x509 -key server_key.pem -out server.pem -days 3650
Step 3. Create trust stores
> keytool -importcert -trustcacerts -keystore clienttruststore.bks -storetype bks -storepass 123456 -file server.pem -provider org.bouncycastle.jce.provider.BouncyCastleProvider -providerpath /home/work/codes/android/cmd/sdk/tools/lib/bcprov-jdk15on-1.56.jar 
> keytool -importcert -trustcacerts -keystore  servertruststore.bks -storetype bks -storepass 123456 -file client.pem -provider org.bouncycastle.jce.provider.BouncyCastleProvider -providerpath /home/work/codes/android/cmd/sdk/tools/lib/bcprov-jdk15on-1.56.jar 
> keytool -importcert -trustcacerts -keystore  servertruststore.jks -storetype jks -storepass 123456 -file client.pem
Step 4. Combine keys and certificates
> openssl pkcs12 -export -inkey client_key.pem -in client.pem -out client.p12
> openssl pkcs12 -export -inkey server_key.pem -in server.pem -out server.p12
Step 5. Convert from pkcs12
> keytool -importkeystore -srckeystore client.p12 -srcstoretype pkcs12 -destkeystore client.bks -deststoretype bks -provider org.bouncycastle.jce.provider.BouncyCastleProvider -providerpath /home/work/codes/android/cmd/sdk/tools/lib/bcprov-jdk15on-1.56.jar 
> keytool -importkeystore -srckeystore server.p12 -srcstoretype pkcs12 -destkeystore server.bks -deststoretype bks -provider org.bouncycastle.jce.provider.BouncyCastleProvider -providerpath /home/work/codes/android/cmd/sdk/tools/lib/bcprov-jdk15on-1.56.jar

Ubuntu18+rabbitvcs:
That's because a bug was introduced in rabbitvcs-core version 0.16-1.1, 
which is the version in the ubuntu repo for 16.10/yakkety. 
The bug causes the "compare to previous revision" action to always export 
the head revision instead of the desired revisions. 
Someone wrote a fix that hasn't been accepted yet: here. 
One easy solution is just to downgrade rabbitvcs-core to 0.16-1 which works fine 
and is available in the ubuntu repo for 16.04/xenial. 
Don't forget to mark the package as held afterwards so software updater doesn't upgrade over it.
sudo apt-add-repository "deb http://us.archive.ubuntu.com/ubuntu xenial universe"
sudo apt install rabbitvcs-core=0.16-1
sudo apt-mark hold rabbitvcs-core
